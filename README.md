# KeyManager

Приложение для управления ключами (WPF + PostgreSQL)

---

## 🛠 Стек технологий
* **Язык:** C# (WPF)
* **База данных:** PostgreSQL
* **Runtime:** .NET 10 SDK

---

## 📋 Требуемый функционал
* **Управление организациями:** Полный цикл CRUD.
* **Управление ключами:**
    * CRUD операции.
    * Блокировка ключей.
* **Фильтрация и поиск:**
    * Фильтрация ключей по организации.
    * Поиск по значению ключа.
* **Экспорт:** Выгрузка списка ключей в формат Microsoft Word (дополнительно).

---

## 🚀 Инструкция по запуску проекта

Для корректной работы приложения убедитесь, что в вашей системе установлены **.NET 10 SDK** и СУБД **PostgreSQL**.

### 1. Подготовка репозитория
Склонируйте проект на локальный компьютер и перейдите в рабочую директорию:
```bash
git clone <ссылка_на_репозиторий>
cd <название_папки_проекта>

```

### 2. Настройка базы данных

Перед запуском необходимо указать параметры подключения к PostgreSQL:

1. Перейдите в директорию `Initialization`.
2. Откройте файл `appsettings.json`.
3. В секции `ConnectionStrings` задайте корректный путь в `DefaultConnection`:
```json
"ConnectionStrings": {
  "DefaultConnection": "Host=localhost; Port=5432; Database=YourDbName; Username=your_user; Password=your_password"
}

```

### 3. Сборка и миграции

Откройте терминал в корневой папке проекта и выполните последовательно:

* **Сборка проекта:**
```bash
dotnet build

```

* **Применение миграций:**
```bash
dotnet run -- migrate

```

### 4. Запуск приложения

После успешного выполнения миграций вы можете запустить проект двумя способами:

* **Через терминал:**
```bash
dotnet run

```

* **Через исполняемый файл:**
Запустите `.exe` файл, находящийся по пути:
`bin/Debug/net10.0/`
